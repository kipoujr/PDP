Με λίγα λόγια

Ο παρών οδηγός ελπίζει να γίνει ένα βοήθημα για όποιον θέλει να μάθει να προγραμματίζει για διαγωνισμούς. Δεν υπάρχει καμμία φιλοδοξία να προσθέσει κάτι στις υπάρχουσες (εξαιρετικές) πηγές που κυκλοφορούν, θέλει μόνο να τις βάλει σε μία σειρά.

Το εγχείρημα αυτό είναι εκ φύσεως ατελές. Μήπως δεν τα είπαμε όλα; Μήπως είπαμε πολλά κι ο αναγνώστης χάθηκε στις λεπτομέρειες; Η προσπάθεια καταβλήθηκε από πολύ έμπειρους (πρώην και νυν) διαγωνιζόμενους, επί πολλά χρόνια, με πολύ μεράκι, πολλές συζητήσεις και πολλές αναθεωρήσεις. Αυτά μπορούμε να τα εγγυηθούμε. Την ποιότητα όμως του τελικού πονήματος μόνο ο αναγνώστης μπορεί να την κρίνει.

Οποιαδήποτε σφάλματα βαραίνουν αποκλειστικά τους συγγραφείς. Ελπίζουμε η ευφυΐα του αναγνώστη να υπερκαλύψει την όποια ανεπάρκεια.

Competitive Programming: From Zero to Hero

Competitive Programming είναι ο προγραμματισμός που στοχεύει στην επίλυση προβλημάτων σε περιβάλλον διαγωνισμού. Ελάχιστος χρόνος για να λύσεις προβλήματα τα οποία είναι σαν τον καλό γρίφο. Φαίνονται δύσκολα, αλλά μόλις εντοπίσεις την τσαχπινιά τους, καταρρέουν.

Γιατί λοιπόν χρειάζεται θεωρία, αφού οι ωραίοι γρίφοι λύνονται αυθόρμητα; Η απάντηση είναι απλή. Αν για ένα γρίφο χρειαστεί να συνδυάσεις 3 έξυπνα κόλπα για να λυθεί, σε ένα πρόβλημα competitive programming θα χρειαστείς 10. Είτε είσαι ικανός να τα λύσεις και τα 10, είτε έχεις λύσει παλιότερα πολλά προβλήματα που θέλανε 3-4 κόλπα, και τώρα παλεύεις να συνδυάσεις 3 τέτοια για να λύσεις το αρχικό. Σε κάθε περίπτωση η μαγεία του γρίφου παραμένει ολόιδια, μόνο τα εργαλεία αλλάζουν.

Σαν τελική σημείωση, πόση σχέση έχει αυτό με την πραγματική πληροφορική; Όση περίπου το να είσαι έξυπνος στη ζωή με το να είσαι ικανός στους γρίφους. Μπορεί και καθόλου, αλλά είναι μια ωραία αρχή.
